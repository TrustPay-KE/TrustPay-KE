🗄️ Database Setup

Step 1: Run SQL Setup

Execute the SQL script to create all tables and set up Row Level Security:

  1. Go to your Supabase project dashboard
  2. Click on "SQL Editor" in the left sidebar
  3. Click "New query"
  4. Copy and paste the contents of supabase-database-setup.sql
  5. Click "Run" to execute the script

⚠️ Important

Make sure to run the entire script without errors. The script creates all necessary tables, indexes, triggers, and RLS policies.

🔐 Authentication Setup

Step 1: Enable Email/Password Auth

  1. Go to "Authentication" in the left sidebar
  2. Under "Auth providers", enable "Email"
  3. Configure email settings (optional but recommended)
  4. Set up email templates for verification and password reset

Step 2: Create Admin User

  1. Go to "Authentication" → "Users"
  2. Click "Add user"
  3. Create an admin account with your email
  4. After creation, go to "Table Editor" → "users" table
  5. Update the user's role to 'admin'
-- Update user role to admin (run in SQL Editor) UPDATE users SET role = 'admin' WHERE email = 'your-admin-email@example.com';

Step 3: Configure Site URL

  1. Go to "Authentication" → "URL Configuration"
  2. Set "Site URL" to your GitHub Pages URL
  3. Set "Redirect URLs" to include your login page
Site URL: https://trustpay-ke.github.io/TrustPay-KE/ Redirect URLs: https://trustpay-ke.github.io/TrustPay-KE/login-supabase.html

📦 Storage Setup

Step 1: Create Storage Buckets

  1. Go to "Storage" in the left sidebar
  2. Click "New bucket" and create these buckets:
Bucket Name Purpose Public Access
kyc_documents ID cards, passports, photos No
proof_files Escrow proof documents No
profile_images User profile pictures Yes

Step 2: Set Storage Policies

For each bucket, create appropriate RLS policies. For the profile_images bucket (public):

-- Allow public read access to profile images CREATE POLICY "Public profile images are viewable by everyone" ON storage.objects FOR SELECT USING (bucket_id = 'profile_images'); -- Allow users to upload their own profile images CREATE POLICY "Users can upload their own profile images" ON storage.objects FOR INSERT WITH CHECK ( bucket_id = 'profile_images' AND auth.uid()::text = (storage.foldername(name))[1] ); -- Allow users to update their own profile images CREATE POLICY "Users can update their own profile images" ON storage.objects FOR UPDATE USING ( bucket_id = 'profile_images' AND auth.uid()::text = (storage.foldername(name))[1] );
